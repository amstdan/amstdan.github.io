# Redis

如果你还没有安装Redis，参考这里安装下载。

# 添加依赖

Spring Data Redis提供了Spring框架和Redis之间的集成。

在pom.xml的<dependencies>中加入：

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-redis</artifactId>
</dependency>

# 配置Redis服务器

src/main/resources/application.properties 添加以下配置:

...
#redis
spring.redis.database=0
spring.redis.host=127.0.0.1
spring.redis.port=6379              # 默认端口是6379
spring.redis.password=              # 连接密码默认为空      
spring.redis.pool.max-active=8      # 连接池最大连接数
spring.redis.pool.max-wait=-1       # 连接池最大阻塞等待时间。负值表示无限制
spring.redis.pool.max-idle=8        # 连接池最大空闲连接数 
spring.redis.pool.min-idle=0        # 连接池最小空闲连接数 
spring.redis.timeout=0              # 连接超时时间（ms）
...

spring.redis.database： 在Redis中，数据库是用一个整数索引标识，而不是数据库名称。默认连接数据库0（默认有16个数据库）。

# 创建缓存配置类

src/main/java/com/example/demo/config/cache/CacheConfig.java

@Configuration
@EnableCaching
public class CacheConfig extends CachingConfigurerSupport {

	@Bean
	public KeyGenerator keyGenerator() {
		return new KeyGenerator() {
			@Override
			public Object generate(Object target, Method method, Object... params) {
				StringBuilder sb = new StringBuilder();
				sb.append(target.getClass().getName());
				sb.append(method.getName());
				for (Object obj : params) {
					sb.append(obj.toString());
				}
				return sb.toString();
			}
		};
	}

	@Bean
	public CacheManager cacheManager(RedisTemplate<?, ?> redisTemplate) {
		CacheManager cacheManager = new RedisCacheManager(redisTemplate);
		return cacheManager;
	}

	@Bean
	public RedisTemplate<?, ?> redisTemplate(RedisConnectionFactory factory) {
		RedisTemplate<?, ?> redisTemplate = new RedisTemplate<String, String>();
		redisTemplate.setConnectionFactory(factory);
		return redisTemplate;
	}
}
  • 开启缓存的方法是添加@EnableCaching到一个使用@Configuration注解的类上。
  • 配置类继承CachingConfigurerSupport来重写方法keyGenerator, cacheManager。
  • 因为缓存是key-value存储，所以缓存方法的每次调用都需要被转换为适合缓存访问的密钥。
  • keyGenerator()使用类名，方法名，参数名来作为缓存方法的key。
  • cacheManager()提供具体缓存实现(RedisCacheManager) -- Redis。

数据存储到Redis的过程中需要进行序列化操作，默认情况下，使用JDK API序列化。因此相关的实体类需要实现Serializable。你也可以定义其它的序列化方法。

src/main/java/com/example/demo/entity/Admin.java

@Entity
public class Admin implements Serializable {
    ...
}

# 缓存注解

# @Cacheable

@Cacheable用来标注方法是能被缓存的。 对于已经将结果存入缓存的方法，在随后的调用中(参数值相同)，结果将直接从缓存中返回，不用调用方法。

在多线程环境中，某些操作可能会同时调用相同的参数。默认情况下，缓存不会锁定任何内容，将可能导致多次计算相同值，这将违背缓存的目的。在这种情况下，需要将sync属性设置true。 此时，只有一个线程计算值，其它线程将被阻塞，直到缓存值被更新。

@Cacheable(cacheNames="admin", sync=true)

# @CachePut

和@Cacheable不同的是，@CachePut每次都会调用方法来更新缓存，无论是否缓存存在。 不要将@CachePut和@Cacheable用在同一个方法上。它们有着不同的行为，用在一起会导致无法预料的结果。

# @CacheEvict

根据一定的条件对缓存进行清空。

# 缓存注解和CRUD操作redis缓存的对应关系

@Cacheable => Save or Query
@CachePut => Save or Update
@CacheEvict => Delete

# @Caching

@Caching让多个同类型注解用在一个方法上。

@Caching(evict = { @CacheEvict("primary"), @CacheEvict(cacheNames="secondary", key="#p0") })

# @CacheConfig

@CacheConfig是一个类级注释，允许共享缓存名称，自定义KeyGenerator, 自定义的CacheManager和自定义的CacheResolver。

即当一个类中的方法拥有相同的缓存属性时，可以在类上加上统一属性配置，简化配置。

# 使用缓存

通常缓存会用在service层。

创建CacheService:
src/main/java/com/example/demo/service/CacheService.java

public interface CacheService {

	public Admin findAdmin(Long id);
	
	public Admin updateAdmin(Long id);
	
	public void deleteAdminCache();
}

创建CacheServiceImpl:
src/main/java/com/example/demo/service/impl/CacheServiceImpl.java

@Service
public class CacheServiceImpl implements CacheService {

	public @Autowired AdminRepository adminRepository;
	
	@Cacheable("admin")
	@Override
	public Admin findAdmin(Long id) {
		System.out.println("Find admin."+" ID=" + id);
		return adminRepository.findOne(id);
	}

	@CachePut("admin")
	@Override
	public Admin updateAdmin(Long id) {
		System.out.println("Update admin."+" ID=" + id);
		Admin admin = adminRepository.findOne(id);
		admin.setEmail("***");
		adminRepository.save(admin);
		return admin;
	}
	
	@CacheEvict(cacheNames = "admin",allEntries=true)
	@Override
	public void deleteAdminCache() {
		System.out.println("Delete all admin cache");
	}
}
  • 因为我们使用的是自定义的key生成算法，缓存跟类名，方法名，参数名有关，所以在CacheServiceImpl中getAdmin()和updateAdmin()相同的参数操作的并不是同一个缓存Entry。如果你使用Spring默认的key生成算法（在CacheConfig中注释掉方法keyGenerate()）。那么在同一个类下，参数相同(包括值)的不同方法操作的将会是同一个缓存Entry。

创建CacheController:
src/main/java/com/example/demo/controller/CacheController.java

@RestController
@RequestMapping("/cache")
public class CacheController {
	
	public @Autowired CacheService cacheService;
	
	@RequestMapping("/admin/{id}")
	public Admin getAdmin(@PathVariable("id") Long id) {
		return cacheService.findAdmin(id);
	}
	
	@RequestMapping("/admin/update/{id}")
	public Admin updateAdmin(@PathVariable("id") Long id) {
		return cacheService.updateAdmin(id);
	}
	
	@RequestMapping("/admin/delete")
	public void deleteAdmin() {
		cacheService.deleteAdminCache();
	}
}

请通过调整不同的key generate 方法和调用顺序来加深对缓存注解的理解。
