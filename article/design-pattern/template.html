模板模式应用场景：某个大的操作可以被分解为许多小的操作，你知道这些小的操作的逻辑执行顺序，但是这些小操作的具体逻辑细节在不同情况下并不相同。这时就可以使用 模板模式，将这个大的操作定义为一个模板。

模板模式关键点：抽象类，final模板方法，抽象逻辑方法，子类实现抽象逻辑方法具体细节，钩子。

public abstract class AbstractClass {

	public final void templateMethod() {
		logicMethodA();
		logicMehodB();
		logicMethodC();
	}
	
	abstract void logicMethodA();
	abstract void logicMethodB();
	//逻辑方法C是钩子
	void logicMethodC() {
		//empty
	}
}

抽象类定义了final模板方法和一些抽象逻辑方法。模板方法中调用了这些抽象逻辑方法，子类通过继承这些抽象逻辑 方法实现具体细节。 钩子是一种被声明在抽象类中的逻辑方法，但是只有空的或默认的实现。通过钩子，可以让子类来决定是否覆盖抽象逻辑方法，是否执行模板方法中的某个逻辑方法。
